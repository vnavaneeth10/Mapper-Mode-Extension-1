# Contributing

Thank you for your interest in contributing to Controlled Queue Loader.

This project focuses on clarity, reliability, and real-world usability. Contributions that improve user experience, stability, or maintainability are welcome.

---

## How to contribute

1. Fork the repository
2. Create a new branch for your change
3. Keep changes focused and minimal
4. Test your changes locally
5. Open a pull request with a clear description

---

## Contribution guidelines

- Keep code readable and well-structured
- Avoid introducing unnecessary complexity
- UI changes should prioritize clarity over visual noise
- Do not add new permissions without strong justification
- Follow existing coding patterns where possible

---

## Reporting issues

If you find a bug or have a suggestion:

- Open an issue with clear reproduction steps
- Explain expected behavior vs actual behavior

---

## Scope

This project is intentionally lightweight.  
Large feature proposals should be discussed via issues before implementation.

---

Thank you for helping improve the project.
